1) Zielgruppe und Zweck

  • Anwender: taegliche Nutzung von Dashboard, Vergleich und Export.
  • Administratoren: Benutzerverwaltung, Rechte, Onboarding, Alert-Regeln.
  • Support: strukturierte Anwenderfuehrung und Fehleranalyse.

2) Schnellstart

  1. .env aus .env.example erstellen.
  2. Domain-, Admin- und PostgreSQL-Werte setzen.
  3. Stack starten: docker compose up -d --build
  4. Aufrufen: https://DEINE_DOMAIN

6) Schritt-fuer-Schritt Bedienung

A) Client auswaehlen und Details pruefen

  1. Navigation: Dashboard.
  2. In der Client-Tabelle auf den gewuenschten Client klicken.
  3. Im Detailbereich Snapshot, Laufwerke, Uptime und Hardware pruefen.
  4. Zeitraum oben umstellen (z. B. 24h / 7 Tage), um Verlaufswerte zu filtern.

B) Vergleich mehrerer Clients

  1. Navigation: Compare.
  2. Clients per Checkbox auswaehlen.
  3. Vergleichswerte pruefen und Auffaelligkeiten identifizieren.
  4. Bei Bedarf zurueck ins Dashboard und Client im Detail pruefen.

C) Inventardaten bearbeiten

  1. Im Dashboard einen Client auswaehlen.
  2. Auf Inventardaten bearbeiten klicken.
  3. Felder ausfuellen (z. B. Standort, Inventarnummer, Verantwortlicher).
  4. Mit Speichern bestaetigen.

D) Export erzeugen

  1. Client im Dashboard auswaehlen.
  2. Exportformat waehlen: JSON, CSV oder PDF.
  3. Datei herunterladen und lokal speichern.

7) Client-Onboarding

  1. Dashboard oeffnen und auf Neuen Client hinzufuegen klicken.
  2. Onboarding-Token erzeugen und Installationsbefehl kopieren.
  3. Befehl auf Zielsystem ausfuehren (Windows/Linux).
  4. Zurueck zur UI: Aktualisieren klicken.
  5. Neuen Client in der Tabelle bestaetigen.

8) Inventarisierung

Verfuegbare Felder: Standort, Inventarnummer, Seriennummer, Abteilung, verantwortliche Person, Lieferant, Anschaffungsdatum, Anschaffungspreis, Garantie bis, Notizen.

9) Alerts & Events

  • Alert-Regeln werden serverseitig gegen Snapshots geprueft.
  • Bei Regelverletzung wird ein Alert-Eintrag erzeugt.
  • Event-Anzeige setzt das Recht view_events voraus.

11) Support-Leitfaeden (Anwenderfuehrung)

Leitfaden 1: "Ich sehe meinen Client nicht"

  1. Anwender auf Dashboard fuehren.
  2. Aktualisieren anklicken lassen.
  3. Filter/Suche pruefen.
  4. Wenn weiter unsichtbar: Agent-Status, Onboarding, Logs pruefen.

Leitfaden 2: "Ich kann keine Events sehen"

  1. Im Dashboard Event-Bereich pruefen.
  2. In Users Rechte des Kontos kontrollieren.
  3. view_events setzen.
  4. Benutzer neu anmelden lassen.

Leitfaden 3: "Export funktioniert nicht"

  1. Client im Dashboard auswaehlen lassen.
  2. Export erneut ausloesen.
  3. Downloadblocker pruefen.
  4. Fehlermeldung dokumentieren und Server-Logs pruefen.

Leitfaden 4: "Benutzer kann nicht deaktiviert werden"

  1. Pruefen, ob es das aktuell angemeldete Konto ist.
  2. Hinweis: Selbst-Deaktivierung ist systemseitig gesperrt.
  3. Aenderung mit anderem berechtigten Konto durchfuehren.

12) Troubleshooting

Traefik liefert 404
  • TRAEFIK_DOMAIN pruefen.
  • docker compose logs -f traefik
Login-/Rechteproblem
  • Benutzer aktiv?
  • Rolle/Rechte korrekt?
  • Self-Deactivation ist bewusst blockiert.
Datenbankauth fehlgeschlagen
  • POSTGRES_PASSWORD pruefen.
  • Bei alten Volumes Migrations-/Passworthistorie beachten.

13) Sicherheit

  • HTTPS/TLS fuer externe Zugriffe
  • Passwortpolicy (mind. 8 Zeichen + Sonderzeichen)
  • Token serverseitig gehasht
  • Rollenbasierte Zugriffskontrolle

14) FAQ

Warum sehe ich keine Events?
Es fehlt vermutlich das Recht view_events.

Warum kann ich mich nicht selbst deaktivieren?
Sicherheitsmechanismus gegen Kontoverlust.

Warum sind Demo-Daten eingeschraenkt?
Trennung von Demo- und Produktivsicht.

Hierbei handelt es sich um ein Tool, das im Rahmen einer Projektarbeit entstanden ist.
